# 院内運用のセキュリティ考慮事項

ローカルLLMを病院・薬局の院内環境で安全に運用するためのガイドです。

> **前提**: 本ガイドはIT管理者・薬剤部責任者と連携して設定することを推奨します。

---

## 基本原則

ローカルLLMの最大のセキュリティ上のメリットは「データが外部に出ない」ことです。  
ただし、設定を誤ると院内LANに不必要な口が開いてしまいます。以下の設定を必ず確認してください。

---

## 1. ネットワーク設定（最重要）

### Ollama のバインドアドレス制限

Ollama はデフォルトで `0.0.0.0:11434` にバインドされます（全インターフェース）。  
**院内LANからアクセスされないよう、必ず localhost に限定してください。**

```bash
# 環境変数で制限（Linux/Mac）
export OLLAMA_HOST=127.0.0.1
ollama serve

# Windowsの場合（PowerShell）
$env:OLLAMA_HOST = "127.0.0.1"
```

本リポジトリの `docker-compose.yml` では、ポートバインドを以下のように設定済みです：

```yaml
ports:
  - "127.0.0.1:11434:11434"  # localhost のみ
  - "127.0.0.1:3000:8080"    # localhost のみ
```

### ファイアウォール設定

| ポート | 用途 | 推奨設定 |
|---|---|---|
| 11434 | Ollama API | 院内LAN外から遮断 |
| 3000 | Open WebUI | 院内LAN内のみ許可（共有利用時） |
| 1234 | LM Studio API | localhost のみ |

---

## 2. モデルファイルの安全な取得

### GGUFファイルのリスクと対策

- **リスク**: 第三者が改ざんしたGGUFファイルには悪意あるコードが含まれる可能性があります
- **対策**:
  - 公式ソースのみ使用（Ollama公式ライブラリ / HuggingFace の開発元公式リポジトリ）
  - sha256 ハッシュを検証する

```bash
# sha256 検証（Windows PowerShell）
Get-FileHash .\model.gguf -Algorithm SHA256

# sha256 検証（Mac/Linux）
sha256sum model.gguf
```

### Modelfile（システムプロンプト設定）のリスク

- **リスク**: 外部から取得した Modelfile にプロンプトインジェクションが仕込まれる場合があります
- **対策**: Modelfile は信頼できるソースからのみ取得し、内容を必ず目視確認してから適用する

本リポジトリの `configs/modelfile_examples/` 内のファイルはすべてオープンソースであり、内容を確認した上でご利用ください。

---

## 3. プロンプトインジェクション対策

外部から取り込んだ文書（PDFテキスト・Webページ等）をLLMに渡す場合、悪意ある指示が埋め込まれている可能性があります。

**対策：**
- システムプロンプトで役割を固定し、ユーザー入力でシステムプロンプトが書き換えられないよう設計する
- 「提供された文書の情報のみを使って回答してください」と制約を明示する
- 出力内容を専門家が必ず確認する

---

## 4. 患者情報の取り扱い

**ローカルLLMであっても、以下は守ってください：**

- 患者氏名・生年月日・患者IDをプロンプトに含めない
- カルテ本文をそのままコピーしてLLMに投入しない
- 「事例」として使用する場合は個人識別情報を必ず匿名化する

> 個人情報保護法・医療機関のプライバシーポリシーに従い、AIツールへの入力可能範囲を事前に確認してください。

---

## 5. 院内共有での運用

薬剤部サーバーや共有PCにOllama + Open WebUIを設置する場合：

```yaml
# docker-compose.yml でのネットワーク設定例（院内LANでの共有）
services:
  open-webui:
    ports:
      - "192.168.1.100:3000:8080"  # 薬剤部サーバーのIPを指定
```

- 院内LANセグメントのみに公開する
- Open WebUI のユーザー管理機能でアクセス制限する
- 院内のIT管理者・情報セキュリティ担当と連携して設定する

---

## 6. 院内運用チェックリスト

```
セットアップ時:
- [ ] Ollamaのバインドアドレスを localhost に限定したか
- [ ] ファイアウォールでポート 11434/3000 を適切に制限したか
- [ ] ダウンロードした GGUF ファイルの sha256 を検証したか
- [ ] Modelfile の内容を目視確認したか

運用時:
- [ ] 患者個人情報をプロンプトに含めていないか
- [ ] AIの出力を専門家が確認してから使用しているか
- [ ] Ollama のバージョンを定期的に更新しているか（セキュリティパッチ）
- [ ] AIを「医療判断の補助」として使用し「医療行為の代替」にしていないか
```
